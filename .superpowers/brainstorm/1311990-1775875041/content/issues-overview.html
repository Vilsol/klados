ResourceList Improvements — Issues & Proposed Fixes

7 issues total: 3 reported + 4 discovered. Review proposed approach for each.

1. Sticky Column Styling

Current (ugly shadow)
nginx-abc12

kube-system Running
Proposed (clean border)
nginx-abc12 default Running
redis-def34 kube-system Running

Fix: Replace bg-bg shadow-[...] with bg-bg border-r border-border. Background inherits row height from the grid row, no more visible gap.

2. Column Resize Jump

The problem & fix

Current (jumps)

startWidth = col.width ?? 100
  ↑ column has no width set
  ↑ actual rendered = 240px
  ↑ snaps to 100 + delta → JUMP

Fixed (smooth)

el = header cell DOM element
startWidth = el.getBoundingClientRect().width
  ↑ always real rendered width
  ↑ delta applies from truth → SMOOTH

Fix: On mousedown, read getBoundingClientRect().width from the header cell. Use that as baseline instead of col.width ?? 100.

4. Horizontal Scroll

Problem: Wide tables get clipped — no way to see columns past the viewport edge.
Fix: Add overflow-x: auto to the scroll container. The grid's min-width is set by the column pixel widths, so it naturally overflows and becomes scrollable.

5. Context Menu Overflow

Problem: Right-click near screen edge → menu renders off-screen.
Fix: After positioning, clamp left and top to keep the menu within the viewport. Measure menu dimensions with getBoundingClientRect() after render.

6. gridTemplateCols Complexity

Problem: String concatenation building the grid template is fragile and hard to read.
Fix: Build as an array of column width strings, then .join(' '). Each column type (checkbox, data, plugin, sparkline, actions) pushes its entries independently.

7. Delete Button Without canMutate Guard

Problem: The fallback delete trash icon shows even when the cluster connection is read-only.
Fix: Wrap the fallback delete button in {#if canMutate}, matching the checkbox column guard.
